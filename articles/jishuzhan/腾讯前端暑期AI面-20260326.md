## 逐题深度解析（摘要）

### 事件循环
- 执行顺序：同步代码 → 清空微任务队列 → 执行一个宏任务 → 再清空微任务队列 → 循环
- 微任务优先级高于宏任务

### HTTPS优势
1. 加密传输（防窃听）
2. 身份验证（防伪装）
3. 数据完整性（防篡改）
4. 性能优势（HTTP/2基于HTTPS）
5. 浏览器信任

### TS的any替代方案
- unknown + 类型守卫
- 联合类型
- 泛型
- Record<string, T>
- 声明模块

### React性能优化
- React.memo / useMemo / useCallback
- 虚拟滚动（react-window）
- 代码分割（React.lazy + Suspense）
- 状态细化
- useTransition
- 列表key优化

### 浏览器缓存
- 强缓存：Cache-Control / Expires
- 协商缓存：ETag / Last-Modified
- 缓存位置：Service Worker → Memory Cache → Disk Cache → Push Cache

### CSS优化减少首屏加载
1. 关键CSS内联
2. 压缩与合并
3. 移除未使用的CSS（PurgeCSS）
4. 避免CSS import
5. 字体优化（font-display: swap）

### 可复用表单组件设计
1. 需求分析
2. 接口设计（props定义）
3. 状态管理
4. 校验实现（同步/异步）
5. 布局适配
6. 无障碍支持
