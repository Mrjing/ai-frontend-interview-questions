1. 简单介绍一下 SSE 和 WebSocket 的区别，为什么 Al 聊天场景多采用 SSE?
难度: 妇女
考点: 网络协议、流式传输
SSE 是一种基于 HTTP 的单向推送协议，相比 WebSocket 更轻量且原生支持断线重连，非常适合
LLM 的流式输出场景。
。 协议层面: SSE 基于标准 HTTP 协议，WebSocket 是独立的 TCP 协议。
。 数据流向: SSE 仅支持服务端向客户端推送，WebSocket 支持全双工双向通信。
。 浏览器支持: SSE 具有内置的自动重连机制和事件 ID 追踪，开发成本较低。
。 场景匹配: LLM 输出是典型的单向持续文本流，SSE 的开销更小且兼容性更好。
常见坑:
。 误认为 SSE 只能传输文本，其实可以通过 Base64 传输二进制。
。 忽略了 HTTP/1.1 下浏览器对同一域名的 6 个并发连接限制。

。 模型生成的参数不符合 Schema，前端缺乏校验逻辑导致崩溃。
。 循环调用问题: 模型不断调用工具进入死循环。
